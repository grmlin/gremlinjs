Triggered, if a gremlin was created, loaded and is ready for use.
This event is triggered on the gremlin's dom element itself and the body of the page
             		                            
    LOAD: "gremlinjs_gremlin_load"
             		           
                               
                            
                 ¶               
               
Triggered, if the content of a gremlin changed.
This event is important, if you changed the HTML code (eg. got some fresh new HTML with Ajax), and need the Loader to rescan the content for new gremlins.
You have to trigger this event manually in your gremlin class though, as there is no good way to detect HTML manipulations in an efficient way in all browsers.



To make the developers life easier, use the gremlinjs/gremlins/AbstractGremlin#triggerChange method
             		                            
    CONTENT_CHANGED: "gremlinjs_gremlin_content_changed"
