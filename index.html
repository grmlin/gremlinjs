GremlinJS, a lightweight and fast widget framework using jQueryD  and requirejsD . Create gremlins gremlinkleines Monster, das Webseiten heimsucht und bevölkert. Gremlins ernähren sich im allgemeinen von Javascript und HTML, werden nie satt und können nur durch Disziplin und Ordnung im Zaum gehalten werden. widgets for your website and let GremlinJS do all the glue work for you.

Fast and clean Javascript

GremlinJS is lightweight, about 4kb minified/zipped, and it's fast. Create jQuery-rich Websites without slowing everything down. All scripts are loaded asynchronously, website loads first, javascript afterwards.

AMD modules

Every gremlin has to be built as a web module. Head over to the requirejs homepage, learn what web modules are and why the Asynchronous Module Definition (AMD) is an awesome way to load your javascript.

Efficient jQuery

There will be only one jQuery selector traversing the dom to discover the gremlins.
Less traversals j better performance.

Uncluttered HTML

Add a classname identifying your gremlins and the gremlin's name to the html-tag of your choice. Filenames, dependencies and object construction are nothing you have to care about.

Notifications and jQuery helper

One Gremlin wants to talk to another? A notification system is built in!
Tired of typing dozens of lines adding jQuery-objects and events to your classes. GremlinJS fixes that.

Lazy loading

You placed some heavy javascript at the end of the page? GremlinJS won't load it until the user scrolls the gremlin into the viewport.

How it works

1. Create the gremlin's javascript module

js/gremlins/HelloWorld.js

2. Add the gremlin to your HTML

<div class="gremlin" data-gremlin-name="HelloWorld"></div>

3. Start the gremlin loader

gremlinjs.getLoader("js/gremlins/","gremlin").load();

That's it

The gremlin loader will find the Gist gremlin in your HTML, loads the file js/gremlins/Gist.js and calls the gremlin's constructor function.

When should I use GremlinJS?

  • You don't like the typical javascript/jQuery mess. GremlinJS forces you to organize your code.
  • You are in a project with tons of javascript widgets acting independently.
  • You are in a project with tons of javascript widgets interacting with each other.
  • You are using a CMS and you hate to scatter the javascript code (inline) across the HTML because you don't know what you'll need on a page and it's not an option to load all the javascript always and everywhere.
  • You are happy if your website responds as quick as possible and jQuery is used efficiently.

What it's not?
GremlinJS is none of those fancy MVC-Frameworks like JavascriptMVC, Spine, Backbone or Sproutcore we love so much to build awesome webapps.
No one is stopping you from using MVC patterns for complex gremlins, of course...

Prerequisites

requirejs

RequireJS is the underlying AMD loader GremlinJS uses to load the gremlin classes. You have to download requirejs from their site, you'll need it later.
Moreover, it's a good idea to learn how requirejs works. They have plenty of articles about requirejs and AMD, so take the time and read them.

jQuery

Currently, GremlinJS uses jQuery to find gremlins in the dom, so go get it.
There may be support for other, more lightweight, frameworks in the future...

Configuration

You have to tell requirejs where it finds jQuery and GremlinJS. For jQuery, GremlinJS expects it with the reference name jquery, as it's provided by newer jQuery versions itself.
GremlinJS consists of multiple modules, the one your'e interested in is called gremlinjs. This module offers you easy access to GremlinJS' core features.
I use the root of the site as the baseUrl for the requirejs config most of the time.

A list of all available GremlinJS modules and path-variables for requirejs can be found in the sources

Add a gremlin

Now we have GremlinJS up and running. It searches the site for all gremlins with the classname gremlin and loads them from the directory js/gremlins/. So what's missing? A gremlin, of course

The HTML

Adding a gremlin to your site is easy.

  1. Add a new element to your html. (this will be the gremlin)
  2. Add a class to that new element identifying it as a gremlin
  3. Add the name of the gremlin

Extending the index.html from above, you'll get this:

index.html
Loading gist...

The classname gremlin is the default classname GremlinJS uses to find gremlins in the site. You can override the default class when you instantiate the Loader. The gremlin's name is set in the data-gremlin-name-attribute.

If you try to load the page now two things will happen.

  1. A HTTP 404 error occurs, because there is no gremlin javascript file with this name
  2. RequireJS throws an exception (with some delay), because it couldn't load the gremlin

The good: something happens and GremlinJS works. The bad: obviously some javascript is missing here.

The Javascript

So let's add the missing file, the gremlin class.

  • /

We told the Loader to use js/gremlins/ as the root path for our gremlins, that's why we have to create a subdirectory called gremlins inside our js directory.

The gremlin's js filename has to be the one you used in the data-gremlin-name-attribute. For the gremlin MyGremlin we added to the HTML before, GremlinJS will try to load a file called js/gremlins/MyGremlin.js

Every gremlin has to be defined as a requirejs-module as it's loaded on demand when it appears on the site.
GremlinJS provides an easy way to create new gremlins, call gremlinjs.create("name",{mixin}) and you're fine.

The gremlin is created using the gremlinjs.create method with two parameters, the name of the gremlin and a mixin extending the basic gremlin class.
initialize will be called, when the gremlin is instantiated. You'll do everything you have to do to get the gremlin ready, in here.

If this HelloWorld gremlin is found the the gremlin's html element will print Hello World!

What we did

  1. Added jQuery, RequireJS and GremlinJS to the project
  2. Created a main.js, that functions as a bootstrap for the site. It creates a GremlinJS Loader and than loads the gremlins in the page.
  3. Created an index.html that includes requirejs using the main.js as the requirejs main file.
  4. Added the gremlin MyGremlin to the index.html and created the corresponding MyGremlin.js requirejs module.
